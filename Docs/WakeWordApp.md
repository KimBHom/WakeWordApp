1. 專案簡介
本系統是一個 可訓練喚醒詞（Wake Word）App，
使用者可以錄製自己的喚詞（3～5 次），
系統在本地訓練並生成專屬模型，
前景服務常駐監聽並在喚詞出現時觸發回應。

特點：

完全離線

不上傳聲音

使用者自訂喚詞

本地訓練

低耗電常駐監聽

3. Android App 架構
3.1 UI 層
喚詞設定頁

錄音頁（顯示音量條、錄製次數）

測試頁（顯示偵測狀態）

3.2 前景服務（Foreground Service）
常駐監聽麥克風

呼叫 MFCC → 模型推論

發送喚醒事件

引導使用者關閉電池最佳化

4. 語音處理層（DSP）
4.1 音訊擷取
AudioRecord

16kHz

Mono

PCM 16-bit

4.2 特徵抽取（MFCC / Log-Mel）
Frame：25ms

Hop：10ms

FFT：512

Mel bins：40

輸出：time_steps × feature_dim（如 49 × 40）

5. 模型層（TFLite / Vosk）
5.1 TFLite（推薦）
模型：CNN / DS-CNN / CRNN

輸入：MFCC

輸出：喚詞 / 非喚詞（二分類）

支援使用者專屬 adapter（小型權重檔）

5.2 Vosk（ASR 路線）
連續語音 → 文字

文字比對喚詞

模型較大、較耗電

6. 本地訓練模組
6.1 訓練流程
使用者錄 3～5 次喚詞

系統加入背景噪音（負樣本）

全部做 MFCC

手機端做輕量訓練：

微調最後一層

或訓練 adapter

或調整 threshold

儲存使用者專屬模型

6.2 儲存內容
使用者喚詞名稱

使用者專屬權重

threshold

7. 喚醒邏輯層（KWS）
每 100ms 推論一次

Sliding Window：最近 1 秒取最大值或平均

Threshold：超過門檻才算喚醒

Debounce：避免連續觸發
