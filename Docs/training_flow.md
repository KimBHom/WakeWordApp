# 🔁 WakeWordApp — 本地訓練流程說明（training_flow）

本文件說明 WakeWordApp 中，使用者如何在裝置上「本地訓練」自己的喚醒詞模型，  
包含錄音流程、資料前處理、特徵抽取、模型訓練與模型更新機制。

---

## 1. 🎯 訓練目標（Training Objective）

- 讓使用者可以「自訂喚醒詞」（例如：嘿小喚、起床了、開導航）
- 全程在本地裝置完成訓練，不上傳任何語音
- 使用少量樣本（3～5 次錄音）即可得到可用模型
- 訓練完成後可立即套用於前景服務（Foreground Service）做即時偵測

---

## 2. 🎙 使用者錄音流程（User Recording Flow）

1. **選擇喚醒詞文字描述（可選）**
   - 例如輸入：「嘿小喚」
   - 僅作為標記與 UI 顯示，不參與訓練

2. **錄製多次樣本**
   - 建議：**3～5 次** 錄音
   - 每次長度約 **1～2 秒**
   - 提示使用者：
     - 在安靜環境錄音
     - 說話音量自然，不要刻意大吼
     - 每次錄音間隔 1 秒以上

3. **錄音完成後進入「處理中」狀態**
   - UI 顯示「正在分析你的喚醒詞…」
   - 背後開始進行音訊前處理與特徵抽取

---

## 3. 🎛 音訊前處理（Audio Preprocessing）

對每一段錄音進行：

- **重採樣（Resample）**
  - 統一為 `16kHz / mono`
- **Pre-emphasis**
  - 強調高頻，提升語音特徵
- **Framing & Windowing**
  - Frame：25ms
  - Hop：10ms
  - Window：Hamming
- **靜音偵測與裁切（VAD / Silence Trimming）**
  - 去除前後長時間靜音
  - 保留主要發音區段
- **（可選）降噪（Denoise）**
  - 簡單的噪音抑制，避免環境噪音影響

---

## 4. 🎚 特徵抽取（Feature Extraction — MFCC）

對處理後的語音做 MFCC 特徵抽取：

- MFCC 維度：**13～40 維**
- 可選：加入 **Δ、ΔΔ**（一階、二階差分）
- 1 秒語音 ≈ 98 個 frame

輸出張量範例：

```text
shape = [98, 40]   # 1 秒 MFCC
dtype = float32
